Cinema REST Service

Cinema REST Service is an API built to be used by cinema theatres. This API allows the user to create a cinema room based on the number of rows and seats per row requested by the user. The seats can be viewed through a /seats GET endpoint and purchased through a /purchase POST endpoint with a seat object body and can be returned through a /return endpoint. Once the user purchases a ticket the stats section of the /seats JSON is updated to show the total earnings and percentage of seats booked.


Skills: Java, Spring Boot, Spring Web, PostgreSQL, RESTful API.


Find out more- https://github.com/kamrantahir2/CinemaRESTService

Once the user has determined how many rows and seats per row they want, a database will be created to keep track of all seats, their price and their booked status.

The user can book seats by sending a POST request to the /purchase endpoint with a seat object as the body. If the seat is available a 200 OK status will be returned and the booked status of the seat will be set to false in the database.
